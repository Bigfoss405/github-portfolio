Technical Artifact: EventBuddy

CS-499 Capstone | Single Comprehensive Artifact

Artifact Overview

Artifact: EventBuddy – Android Mobile Application
Original Course: CS-360 Mobile Architecture & Programming
Purpose: EventBuddy allows users to create, edit, and manage personal events using a local SQLite database within an Android application.
Technology Stack: Java, Android SDK, SQLite, RecyclerView, Android Studio
Capstone Focus Areas: Software Design & Engineering, Algorithms & Data Structures, and Databases

Enhancements Completed

Enhancement 1: Software Design and Architecture Improvements

  • Refactored core components such as the DBHelper, event model, and UI activities to improve separation of concerns and modularity.
  • Improved workflow between application screens to ensure consistent state management and smoother user interactions.
  • These changes improved code readability, maintainability, and overall application structure.

Enhancement 2: Algorithms and Data Handling Optimization

  • Improved how event data is retrieved, sorted, and passed between activities.
  • Optimized RecyclerView adapter lifecycle handling to ensure efficient and reliable UI updates.
  • Enhanced event object processing to improve application responsiveness when working with structured data.

Enhancement 3: Database Design and CRUD Improvements

  • Refined the SQLite database schema to support more consistent and structured event storage.
  • Improved CRUD operations to ensure accurate, efficient, and secure data handling.
  • Strengthened validation and interaction between the database layer and the user interface.
